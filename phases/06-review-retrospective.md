# Phase 6: Review & Retrospective

## Purpose

The final phase is about evaluation and learning. You're answering: **Did we succeed? What went well? What could be better?**

This phase focuses on:
- Evaluating the outcome against requirements
- Capturing learnings and insights
- Identifying improvements for next time
- Celebrating successes
- Planning for maintenance and evolution

## Key Questions

The Review Agent will help you explore:

### Outcome Evaluation
- Did we meet all requirements?
- Are acceptance tests passing?
- Does it work in production?
- Are users satisfied?

### Quality Assessment
- Is the code maintainable?
- Are there technical debt items?
- What architectural decisions held up?
- What would we change?

### Process Reflection
- Did Loom framework work?
- Which phases were most valuable?
- Where did we struggle?
- What was smooth?

### Team Learning
- What did the team learn?
- What surprised us?
- What do we want to remember?
- What will we do differently next time?

### Future Planning
- What needs maintenance?
- What's the roadmap?
- What technical debt needs addressing?
- How do we scale this?

## Outputs

By the end of Review & Retrospective, you'll have:

✅ **Outcome Report** - Requirements vs. reality
✅ **Success Metrics** - How did we perform?
✅ **Retrospective Notes** - What we learned
✅ **Action Items** - Improvements for next project
✅ **Maintenance Plan** - How we'll keep this running
✅ **Lessons Learned Document** - Knowledge capture for the team

## When Review is Complete

You've successfully completed Loom when:

- ✅ Requirements are met (or deliberately deferred)
- ✅ Tests are passing
- ✅ Code is deployed/released
- ✅ Team has reflected on the process
- ✅ Learnings are captured
- ✅ Improvements are planned

## The Retrospective Process

### 1. Did We Meet Requirements?
Go back to the Requirements phase:
- Is each requirement implemented?
- Do acceptance tests pass?
- Are there unmet requirements?
- What's intentionally deferred?

### 2. What Went Well?
Celebrate successes:
- What processes worked smoothly?
- What technical decisions paid off?
- Where did we collaborate well?
- What should we do again?

### 3. What Could Be Better?
Identify improvements:
- Where did we struggle?
- What was unexpected?
- What would we change?
- What do we need to learn?

### 4. How Do We Improve?
Create action items:
- Specific improvements to try next project
- Process changes
- Skill development
- Technology decisions to revisit

### 5. Knowledge Capture
Document learnings:
- Update templates based on learnings
- Record architectural insights
- Capture best practices
- Note what surprised you

## Tips for Effective Retrospectives

- **Be honest** - Acknowledge what didn't work
- **Celebrate wins** - Recognize successes
- **Focus on improvement** - Retrospectives are about learning, not blame
- **Be specific** - "We need better testing" is vague; "We should write integration tests earlier" is specific
- **Plan to act** - Identify 2-3 key improvements to apply next project
- **Document it** - Written retrospectives help you remember and share learnings

## Continuous Improvement

Each project teaches you something. Over time:
- You'll refine which phases take longest
- You'll discover best practices for your team
- You'll improve estimates
- You'll build better templates
- The framework will evolve with your needs
